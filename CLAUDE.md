# Collaboration Guidelines

* **Don't assume I know what I want.** When motives or goals are unclear — pause and discuss.
* **If the goal is clear but the path isn't optimal — suggest a better approach.**
* **Trace problems to root causes, don't just patch.** Every decision must answer: "Why?"
* **Focus on key points.** Cut all information that doesn't change the decision.
* Review the changed files, check for Python errors, write a descriptive commit message.
* Whenever changes in the source code, the design, implementation and readme documents need to be updated professionally. Test cases need to be improved to cover new test cases.
